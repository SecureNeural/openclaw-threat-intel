# Start Here

<div class="hero-shell">
  <span class="hero-kicker">Operator guide</span>
  <h1 class="hero-title">How to use OpenClaw Threat Intel</h1>
  <p class="hero-lead">
    This site is built for two common workflows: fast situational awareness and
    evidence-backed follow-up. Start with the path that matches your question.
  </p>
</div>

## If you want the latest news

<div class="steps-list">
  <div class="step-card"><strong>1</strong> Open <a href="newsfeed/">Newsfeed</a>.</div>
  <div class="step-card"><strong>2</strong> Scan the newest entries by type: finding, control, or source.</div>
  <div class="step-card"><strong>3</strong> Open the linked canonical finding if the item looks relevant to your environment.</div>
</div>

## If you want the answer, not the article

<div class="steps-list">
  <div class="step-card"><strong>1</strong> Go to <a href="findings/">Findings</a>.</div>
  <div class="step-card"><strong>2</strong> Pick the category: vulnerabilities, exposures, incidents, or malicious-skills.</div>
  <div class="step-card"><strong>3</strong> Read the canonical entry for the summary, impact, and mitigation.</div>
</div>

## If you need proof or original context

<div class="steps-list">
  <div class="step-card"><strong>1</strong> Open <a href="sources/">Sources</a>.</div>
  <div class="step-card"><strong>2</strong> Choose the evidence type: article, docs, post, repo, or advisory.</div>
  <div class="step-card"><strong>3</strong> Follow the source record to the original external material.</div>
</div>

## If you want defensive guidance

<div class="steps-list">
  <div class="step-card"><strong>1</strong> Open <a href="controls/">Controls</a>.</div>
  <div class="step-card"><strong>2</strong> Choose hardening, sandboxing, detection, or scanners.</div>
  <div class="step-card"><strong>3</strong> Use the linked findings to understand which threats each control addresses.</div>
</div>
